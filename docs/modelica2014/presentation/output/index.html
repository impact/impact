Introduction

Package managers are essential components in all modern languages

  • Java - Maven

  • Scala - SBT

  • Python - pip and easy_install

  • Go - integrated into the command line compiler

  • NPM (node package manager) - Node.js (back-end)

  • Bower - Javascript (front-end)

  • Modelica - impact

History

  • No structured central repository of Modelica libraries

  • No "updates available" capability in tools

  • Long standing proposal to define a repository contents format...never went anywhere

Inspiration

Listening to a podcast about Bower

Key take aways:

  • They wrote the core in a few days

  • They leveraged GitHub to manage content

  • They used semantic versioning

Additional Use Cases

Tool Integration

  • Goes beyond previous repository format proposal to create an actual repository

  • Reference implementation in Python is both a tool and a library

  • For non-Python platforms, index.json file is available for all implementations via modelica.org

Private Repositories

  • Currently supported:

    • Via private indices
    • Private index needs to be privately hosted
    • Repositories must private GitHub repositories
    • OAuth tokens and configurations files

  • Goal:

    • Support other repository types (git, svn)
    • Simpler and streamlined

Automation

  • Mechanism to support continuous integration involving Modelica models (Jenkins/Hudson)

  • Simulation services

Future Work

Potential improvements

  • Support for a "package.json" file

    ...to root level dependencies like npm or bower

  • An upgrade sub-command

    ...to check for updates

  • Support additional repository types

    ...git, svn, etc.

  • Pull requests welcome!

Conclusions

  • Command-line tool and Python library

  • Open source license

  • Completely cross platform (Windows, Mac, Linux)

  • Handles dependencies

  • Index of public packages hosted on modelica.org

  • Syncronized with modelica.org/libraries
